# Sprint-Review-and-Retrospective
How do I interpret user needs and implement them into a program? How does creating “user stories” help with this?

To understand user needs, I first focus on what the user is trying to accomplish. I ask what problem they are solving and what result they expect. Then I break that into small features the program needs to have.

User stories help because they clearly explain who the feature is for and why it matters. Writing them in the format “As a user, I want ___ so that ___” keeps the focus on the user instead of just the code. It makes it easier to plan and prioritize what to build first.

How do I approach developing programs? What Agile processes do I hope to use in the future?

I develop programs step by step instead of trying to build everything at once. I break big projects into smaller tasks and complete them over short periods of time.

In this course, I used Agile practices like sprint planning, reviews, and retrospectives. In the future, I want to keep using short sprints, regular check-ins, and feedback sessions. These help teams stay organized and improve over time.

What does it mean to be a good team member in software development?

Being a good team member means communicating clearly, finishing your work on time, and helping others when needed. It also means being open to feedback and working toward the same goal as the team.

Software development works best when everyone collaborates and supports each other.
